Security Modes

Vulnerable Mode

Resolvers use raw SQL string concatenation, no authentication checks, unrestricted file uploads, and full error stack traces. This is the default mode for exploitation practice.

Mitigated Mode

Resolvers use parameterized queries, enforce JWT authentication and ownership checks, validate file types, and sanitize error messages. Toggle this mode to see the secure implementation.

Architecture

GraphQL API

Single /graphql endpoint powered by Apollo Server 4. All queries and mutations go through this unified interface.

PostgreSQL

Direct pg client with raw SQL queries. Vulnerable mode uses string concatenation; mitigated mode uses parameterized queries.

JWT Auth

JSON Web Token authentication via Express middleware. Vulnerable mode accepts alg: none; mitigated mode enforces HS256 signature verification.
